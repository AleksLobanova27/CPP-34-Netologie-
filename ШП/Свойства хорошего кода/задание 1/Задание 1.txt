1. Нарушение принципа подстановки Барбары Лисков (LSP):

Класс Data нарушает LSP, потому что его методы printAsHTML(), printAsText() и printAsJSON() бросают исключения, если формат не соответствует ожидаемому.

Клиентский код, работающий с объектом через базовый интерфейс Printable, ожидает, что любой объект Printable сможет быть преобразован в любой формат, но объект Data с Format::kText не может быть преобразован в HTML или JSON, что нарушает контракт базового класса.

2. Нарушение принципа открытости/закрытости (OCP):

Код не открыт для расширения, а это означает что при добавлении нового формата (например, XML) придётся добавить новое значение в enum class Format, добавить новый виртуальный метод в интерфейс Printable, реализовать этот метод во всех существующих классах, добавить новый case в функцию saveTo, добавить новую функцию-обёртку. А так же класс Data привязан к конкретному формату через поле format_ и проверки в каждом методе

3. Нарушение принципа разделения интерфейса (ISP):

Интерфейс Printable содержит методы для всех возможных форматов вывода. Класс, который хочет поддерживать только один формат всё равно вынужден реализовывать все три метода или бросать исключения.
